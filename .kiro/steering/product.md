# Product Overview

Driver Copilot（コード名 **TruckCopilot**）は、長距離トラックドライバーの**運転中の意思決定を支援する音声ファースト型ナビゲーションアプリ**。法定休憩ルール（拘束時間・連続運転）を意識したルート提案と、SA/PA の混雑・駐車場空き予測を軸に「いつ・どこで休むか」の判断を肩代わりすることを狙う。

## Core Capabilities

- **休憩レコメンド付きナビゲーション** — 現在地・目的地・残り連続運転時間から SA/PA 候補をスコアリングし、推奨 1 件＋代替案＋回避理由を返す（`/api/v1/navigation/recommend-rest`）。
- **連続運転タイマー** — 走行/停止を検知し、次の法定休憩までの残時間を常時提示する（`features/driving-timer`）。
- **ハンズフリー音声操作** — `@react-native-voice/voice` による音声コマンド入力と `expo-speech` / VOICEVOX による読み上げで、画面注視を不要にする（`features/voice`）。
- **縦型ルートタイムライン UI** — 運転中でも理解しやすい縦スクロール・タブ切替式のルート表示（`RouteTimeline` / `VerticalRouteSchematic` / `RouteTabListHeader`）。
- **マーケティング・ピッチ素材生成** — Remotion ベースの動画プロジェクト（`video/`）でデモ動画・LP 用素材を作成する。

## Target Use Cases

- 長距離・中継輸送のプロドライバーが、走行中にハンズフリーで次の休憩ポイントを判断したい。
- 運行管理者・事業者が法定休憩ルールの遵守を現場に落とし込みたい。
- 展示会・ピッチ・社内説明でアプリ挙動を動画・モックで提示したい（Remotion コンポジション）。

## Value Proposition

- **運転行為を邪魔しない UI／UX** — 音声入出力と縦型ビジュアルを最優先し、タップ深度とテキスト量を抑える。
- **“守らせる” ではなく “選ばせる”** — 候補と理由（`reason` / `avoidedReasons`）を併記し、ドライバーの最終判断を尊重する設計。
- **Spec-Driven + エージェント駆動開発** — `.kiro/specs/` と Kiro スキル群により、要件→設計→タスク→実装を一貫したワークフローで積み上げる。

## Design Source of Truth
