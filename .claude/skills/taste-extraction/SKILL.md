# taste-extraction（テイスト抽出）

## 目的

単一の「正」ソースが無い場合でも、**参照素材から観測可能な事実**を列挙し、`docs/uiux/theme.md`（トークン・トーン）と `docs/uiux/design-system.md`（コンポーネント規約）に**重複なく**反映する。推測は **推測** とラベルし、出所と紐づける。

## 入力タイプと取得方法

| 種別 | 取得のヒント | 抽出しやすい要素 |
|------|----------------|------------------|
| **URL** | ページ本文・スタイルガイド・マーケ LP。取得は許諾・robots・ToS を確認し、公開情報に限定。 | ヒーロー配色、見出しフォントの言及、CTA の形、余白の広さ |
| **PDF** | ブランドブック、スライド、UI仕様PDF。テキスト抽出＋主要ページの視覚確認（表紙・カラーページ）。 | カラーコード表、タイポスケール、コンポーネント例 |
| **動画** | プロモ・デモ収録・画面録画。代表フレームのキャプチャ（タイムコード）、ナレーション・テロップのフォント雰囲気。 | モーションのテンポ、角丸・影の有無、ダーク/ライト傾向、情報密度 |
| **静止画** | スクショ、モック、Figma 書き出し | パターン（カード、ナビ）、アイコン線幅 |
| **既存コード** | `tailwind.config`、CSS 変数、コンポーネントライブラリ | 実装とドキュメントの乖離修正 |

**動画のフレーム取得例（環境に `ffmpeg` がある場合）**

```bash
ffmpeg -ss 00:00:03 -i input.mp4 -vframes 1 -q:v 2 frame_003.png
```

タイムコードはソース表に記載する。

## 抽出パイプライン

1. **ソース登録**  
   各入力に `S01` のような ID を振り、`docs/uiux/theme.md` / `design-system.md` 先頭の「抽出ソース」表に 1 行追加（種別・参照・日付・備考）。

2. **観測ログ（内部メモ可）**  
   列挙のみ: 色（近似HEX可）、フォント名、角丸、影、グリッド、主要コンポーネントの見た目、モーションの速さ。  
   分からない項目は空欄。矛盾はソース ID ごとに併記。

3. **正規化 → トークン案**  
   - **theme.md**: ブランド一言、カラーテーブル、タイポ、スペーシング、モーション、アクセシビリティ方針。  
   - **design-system.md**: 原則、レイアウト、コンポーネント一覧、代表コンポーネントの anatomy/states、フォーム、アイコン、トークン対応表。

4. **衝突解決**  
   複数ソースで数値が違う場合は、優先順位を人間に確認するか、「プライマリソース: S02」と明記して採用理由を1行書く。

5. **著作権・ライセンス**  
   フォント名・商用利用条件、画像・動画の再掲載可否をメモ。ドキュメントにライセンス不明の素材仕様を断定で書かない。

## ドキュメント役割分担（重複禁止の目安）

| 内容 | 主担当ファイル |
|------|----------------|
| 色・タイポ・余白・モーションの「値」 | `theme.md` |
| コンポーネントの種類・状態・レイアウトルール・フォーム順序 | `design-system.md` |
| CSS/Tailwind 変数名の対応 | `design-system.md` §トークン対応（theme の値を参照） |

## アウトプット必須

- 更新後の `theme.md` と `design-system.md` の差分要約
- 抽出ソース表の更新
- 推測で埋めたセルには「（推測: Sxx）」と注記

## アンチパターン

- ソース無しのブランドコピーだけでトークンを埋める
- theme と design-system に同じ表を二重管理する（一方は「参照」に留める）
- スクレイピングで非公開ダッシュボード等に触れる
