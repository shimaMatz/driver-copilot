# taste-extraction（テイスト抽出）

## 目的

参照素材から観測可能な事実を列挙し、**リポジトリ直下の `DESIGN.md` 1 本**にまとめる。推測は **推測** とラベルし、出所（ソース ID）と紐づける。`DESIGN.md` は [awesome-design-md](https://github.com/VoltAgent/awesome-design-md) と同様の章立て（§1 Visual Theme … §9 Agent Prompt Guide）を前提にし、**同じ情報を別ファイルに分割しない**。

## 入力タイプと取得方法

| 種別 | 取得のヒント | 抽出しやすい要素 |
|------|----------------|------------------|
| **URL** | ページ本文・スタイルガイド・マーケ LP。取得は許諾・robots・ToS を確認し、公開情報に限定。 | ヒーロー配色、見出しフォントの言及、CTA の形、余白の広さ |
| **PDF** | ブランドブック、スライド、UI仕様PDF。テキスト抽出＋主要ページの視覚確認（表紙・カラーページ）。 | カラーコード表、タイポスケール、コンポーネント例 |
| **動画** | プロモ・デモ収録・画面録画。代表フレームのキャプチャ（タイムコード）、ナレーション・テロップのフォント雰囲気。 | モーションのテンポ、角丸・影の有無、ダーク/ライト傾向、情報密度 |
| **静止画** | スクショ、モック、Figma 書き出し | パターン（カード、ナビ）、アイコン線幅 |
| **既存コード** | `tailwind.config`、CSS 変数、コンポーネントライブラリ | 実装とドキュメントの乖離修正 |

**動画のフレーム取得例（環境に `ffmpeg` がある場合）**

```bash
ffmpeg -ss 00:00:03 -i input.mp4 -vframes 1 -q:v 2 frame_003.png
```

タイムコードは抽出ソース表に記載する。

## 抽出パイプライン

1. **ソース登録**  
   各入力に `S01` のような ID を振り、`DESIGN.md` 内の **抽出ソース**表に 1 行追加（種別・参照・日付・備考）。表は由来の引用ブロックの直後、またはファイル末尾の「## 10. Extraction sources（プロジェクト）」など、固定位置に置く。

2. **観測ログ（内部メモ可）**  
   列挙のみ: 色（近似HEX可）、フォント名、角丸、影、グリッド、主要コンポーネントの見た目、モーションの速さ。  
   分からない項目は空欄。矛盾はソース ID ごとに併記。

3. **正規化 → `DESIGN.md` へマージ**  
   - **§1〜§3**: トーン、カラーパレット、タイポグラフィ。  
   - **§4〜§6**: コンポーネント、レイアウト、深度・影。  
   - **§7〜§8**: 禁止事項、レスポンシブ。  
   - **§9**: エージェント向けプロンプト断片があれば更新。  
   awesome 由来のベース本文を残したまま上書きする場合は、「（プロジェクト: Sxx）」などで差分を明示する。

4. **衝突解決**  
   複数ソースで数値が違う場合は、優先順位を人間に確認するか、「プライマリソース: S02」と明記して採用理由を1行書く。

5. **著作権・ライセンス**  
   フォント名・商用利用条件、画像・動画の再掲載可否をメモ。ドキュメントにライセンス不明の素材仕様を断定で書かない。

## セクション対応の目安（1 ファイル内の役割分担）

| 内容 | おおよその節 |
|------|----------------|
| ムード・哲学・密度 | §1 Visual Theme & Atmosphere |
| 色と意味付け | §2 Color Palette & Roles |
| フォント・階層 | §3 Typography Rules |
| ボタン・カード・入力・ナビ等 | §4 Component Stylings |
| 余白・グリッド | §5 Layout Principles |
| 影・面の重なり | §6 Depth & Elevation |
| アンチパターン | §7 Do's and Don'ts |
| ブレークポイント・タップ領域 | §8 Responsive Behavior |
| 実装プロンプトの短文例 | §9 Agent Prompt Guide |

## アウトプット必須

- 更新後の `DESIGN.md` の差分要約
- 抽出ソース表の更新
- 推測で埋めた箇所には「（推測: Sxx）」と注記

## アンチパターン

- ソース無しのブランドコピーだけでトークンを埋める
- `theme.md` / `design-system.md` に情報を分割して二重管理する（本リポジトリでは廃止）
- スクレイピングで非公開ダッシュボード等に触れる
