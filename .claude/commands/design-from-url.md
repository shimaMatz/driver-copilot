# /project:design-from-url

## 目的

人間が**URL をチャットに貼る・リンクとして添付**したとき、そのページを根拠に **`DESIGN.md`（リポジトリ直下）** を [awesome-design-md 型の章立て](https://github.com/VoltAgent/awesome-design-md) で新規作成するか、既存内容へマージする。

## 推奨サブエージェント / スキル

- **design-md-author**（本コマンドの主担当）
- **`taste-extraction`**（単一ファイルへの反映ルール・抽出ソース表）

## 手順

1. 対象 URL を確認する（複数ある場合はソース表に並べるか、優先順を人間に確認）。
2. **design-md-author** の手順に従い、ページを取得 → 観測ログ → `DESIGN.md` の §1〜§9 を埋める。
3. 既存 `DESIGN.md` がある場合は、由来ブロック直下（または定位置）の**抽出ソース**表に今回の URL を `Sxx` で追加し、数値・ルールの衝突は注記で解消する。
4. 取得失敗時（ログイン壁・ブロック）は、スクショ／PDF 提出を依頼し、`/project:extract-taste` に切り替える案内を返す。

## 完了条件

- [ ] `DESIGN.md` が §1〜§9 のいずれか適切な粒度で埋まっている（空のプレースホルダだけにしない）
- [ ] ソース URL と取得の前提（推測箇所）が分かる
- [ ] 公開取得の範囲内で実施したことが守れている

## 注意

- 自動取得は**公開ページ**に限定。社内 URL はエクスポート素材で対応。
- HTML が空や極端に短い場合は CSR の限界を明記し、観測できた範囲だけを書く。
