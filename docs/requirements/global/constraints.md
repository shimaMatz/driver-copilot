# 制約事項

## 技術的制約

### TC-1: Expo SDK の制約

- **ネイティブモジュール制限**: Expo Go 上では一部のネイティブモジュールが動作しない。`@react-native-voice/voice`（音声認識）は Expo Go 非対応であり、開発ビルド（`expo run:ios` / `expo run:android`）が必要。
- **Expo SDK バージョン**: SDK 54 を使用。Expo のアップデートサイクルに依存し、React Native のバージョンも Expo が指定するものに従う（現在 0.81.5）。
- **OTA更新**: Expo のアップデート機構を利用可能だが、ネイティブモジュールの変更時はストア再提出が必要。

### TC-2: Google API 依存とフォールバック

- **Google Maps Platform**: Directions API, Places API, Nearby Search API を使用。従量課金制のため、利用量に応じたコストが発生する。
- **APIキー未設定時**: OSRM（ルート検索）、Nominatim（場所検索）、Overpass API（POI検索）へ自動フォールバックする。これらは無料だが、以下の制限がある。
  - OSRM: 有料道路/一般道の区別不可。ルートステップ情報なし。
  - Nominatim: 使用ポリシーにより秒間1リクエスト制限。User-Agent 必須。
  - Overpass API: クエリが重いとタイムアウトの可能性。
- **レート制限**: 全APIリクエストに 10秒タイムアウトを設定済み。

### TC-3: オフライン動作

- **現状未対応**: ネットワーク接続が必須。`@react-native-community/netinfo` でネットワーク状態を検知する機構はあるが、オフライン時のキャッシュやローカルルート計算は未実装。
- **将来課題**: オフラインマップタイルのキャッシュ、直前のルート情報のローカル保存を検討。

### TC-4: バックエンド依存

- **休憩推薦API** (`/api/v1/navigation/recommend-rest`): 自前バックエンドが必要。未実装時はデモモードのモックデータで代替する。
- **バックエンドURL**: デフォルトは `http://192.168.1.2:8080`（ローカル開発用）。本番環境では `EXPO_PUBLIC_API_BASE_URL` で指定する。

## ビジネス制約

### BC-1: 法規制

- **430ルール**: 貨物自動車運送事業法に基づく連続運転時間の制限（4時間以内に30分以上の休憩）。アプリはこのルールの遵守を支援するが、法的責任はドライバー及び運行管理者にある。
- **運転中の操作制限**: 道路交通法により運転中のスマートフォン操作は禁止。音声操作を主たるインタラクション手段とし、画面タッチ操作は停車中に限定する設計とする。

### BC-2: 地理的スコープ

- **日本国内限定**: 有料道路判定の正規表現、Places API の国コード指定 (`country:jp`)、Nominatim の国コード (`countrycodes: 'jp'`) など、日本向けにハードコードされている箇所がある。
- **多言語対応**: 現状は日本語のみ。API リクエストの `language: 'ja'` 指定。

### BC-3: コスト

- **Google Maps Platform**: 月額の無料枠（$200/月）を超えると従量課金。ルート検索・場所検索が主なコストドライバー。
- **フォールバックAPI**: 無料だが、Nominatim / Overpass は寄付ベースの運営であり、大量リクエストは控える必要がある。

## セキュリティ制約

### SC-1: APIキー管理

- Google Maps API キーは環境変数 (`EXPO_PUBLIC_GOOGLE_MAPS_API_KEY`) で管理する。
- `EXPO_PUBLIC_` プレフィックスにより、クライアントバンドルに含まれる。APIキーには Google Cloud Console 上でアプリ制限（パッケージ名/バンドルID）を設定すること。
- `.env` ファイルはリポジトリにコミットしない。`.env.example` のみコミット対象。

### SC-2: ユーザーデータ

- 位置情報はリアルタイムのナビゲーション目的でのみ使用し、サーバーへの永続保存は休憩推薦リクエスト時の始点・終点座標に限定する。
- AsyncStorage に保存するデータは端末ローカルのみ。
