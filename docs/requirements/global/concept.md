# プロジェクトコンセプト

## ビジョン

**Driver Copilot (TruckCopilot)** は、長距離トラックドライバーの安全と生活の質 (QOL) を向上させるモバイルナビゲーションアプリである。

## 解決する課題

### 1. 430ルール遵守の負担

貨物自動車運送事業法に基づく「430ルール」（連続運転4時間以内、30分以上の休憩）は、ドライバーが自身で時間管理を行う必要がある。長時間の運転中に残り時間を正確に把握し続けることは認知的負荷が高く、違反リスクや安全上の問題につながる。

本アプリは運転時間を自動で計測し、休憩タイミングを音声で通知することで、ドライバーが運転に集中できる環境を提供する。

### 2. 休憩場所の最適化

長距離ドライバーにとって、大型車が駐車可能で混雑していない休憩場所を走行中に探すことは困難である。特に深夜帯のSA/PA混雑は深刻な問題であり、休憩を取れないまま運転を続けるケースが発生する。

本アプリはルート上の休憩候補地を混雑予測付きで提示し、最適な休憩計画を推薦する。

### 3. ドライバーのQOL向上

長距離運転において、銭湯・24時間レストラン・コインランドリーといった生活インフラへのアクセスは、ドライバーの健康と満足度に直結する。しかし、大型車で立ち寄れる施設を効率的に探す手段は限られている。

本アプリはルート周辺のQOLスポットを検索し、立ち寄り候補として提示する。

## コア機能

| 機能 | 概要 |
|------|------|
| ナビゲーション | 有料道路/一般道切替対応のルート検索・地図表示 |
| 運転タイマー | 430ルール準拠の連続運転時間管理・休憩アラート |
| 休憩推薦 | ルート上の休憩スポットを混雑予測付きで推薦 |
| QOLスポット検索 | 銭湯・24時間レストラン・コインランドリーの周辺検索 |
| 音声操作 | ハンズフリーでの目的地設定・情報読み上げ |
| デモモード | APIキー不要で全機能を体験できるデモ環境 |

## ターゲットユーザー

- 長距離トラックドライバー（主に日本国内）
- 運行管理者（将来的な拡張対象）

## 設計方針

- **安全第一**: 運転中の操作を最小化し、音声操作と自動通知を重視する。
- **オフライン耐性**: Google API 未設定時でもオープンソースAPI（OSRM, Nominatim, Overpass）へのフォールバックにより基本機能が動作する。
- **段階的導入**: デモモードにより、バックエンド未構築・APIキー未取得の段階でもアプリの価値を検証できる。
