## 1. 背景と目的

長距離トラックドライバーは、法令遵守（連続運転4時間ルール=430ルール）と休憩場所の確保（大型車枠の空き不足）という日常的な課題を抱えている。

本アプリは、ナビゲーション・運転タイマー・休憩所推薦を統合し、ドライバーが430ルールを遵守しながら効率的に休憩場所を選択できるようにする。MVPでは、安全運行に直結するコア機能に絞り、最小限の依存で動作する構成とする。

## 2. スコープ

### In Scope（MVP = v0.1）

- 地図表示・現在地追従（react-native-maps / expo-location）
- 目的地検索（テキスト入力、Google Places API / Nominatim フォールバック）
- 2ルート比較（一般道 / 有料道路のポリライン描画、距離・所要時間表示）
- ルートIC縦一列表示（RouteTimeline コンポーネント、JR東日本アプリ風UI）
- 430タイマー（連続運転時間・累積休憩時間・残り運転可能時間、5分単位更新）
- 休憩所推薦（SA/PA基本一覧、混雑度・空き台数の表示）
- デモモード（standard シナリオ1本のみ、API不要での基本動作確認）
- Expo SDK ベースのクロスプラットフォーム対応（iOS / Android）

### Out of Scope（MVP外）

- 音声コマンド操作（ネイティブモジュール依存が重く Expo Go 非対応、v0.2 候補）
- デモモード複数シナリオ（430限界直前・SA全滅・QOL優先・本末転倒防止の4シナリオ、v0.2 候補）
- QOLスポット検索（銭湯・24hレストラン・コインランドリー、v0.2 候補）
- 本末転倒防止ロジック（休憩地が目的地から大きく逆行する場合の警告、v0.2 候補）
- 代替休憩場所提案（SA全滅時のコンビニ・道の駅提案、v0.2 候補）
- ユーザー認証・アカウント管理（将来フェーズ）
- 運行日報・勤怠連携（将来フェーズ）
- リアルタイム車両間通信（将来フェーズ）
- 課金・サブスクリプション（将来フェーズ）
- バックエンドサーバーの実装（既存 API を利用）

## 3. ユーザーとゴール

| ユーザー | 達成したいこと |
|----------|----------------|
| 長距離トラックドライバー | 430ルールを遵守しながら、効率的に休憩地を選び、安全に運行を遂行したい |

## 4. ユースケース / ユーザーストーリー

1. **として** ドライバー **したい** 目的地をテキストで設定し、一般道/有料道路の2ルートを比較して出発する **なぜなら** 運行条件に最適なルートを素早く選びたいから
2. **として** ドライバー **したい** 連続運転時間と残り運転可能時間をリアルタイムで確認する **なぜなら** 430ルール違反を未然に防ぎたいから
3. **として** ドライバー **したい** ルート上のSA/PAの混雑状況と空き台数を見て、最適な休憩場所を選ぶ **なぜなら** 大型車枠が満車で休憩できないリスクを避けたいから
4. **として** ドライバー **したい** 430限界が近づいたとき視覚的な警告を受ける **なぜなら** 集中運転中に時間を見落とすリスクがあるから
5. **として** ドライバー **したい** ルート上のIC一覧を縦一列で確認する **なぜなら** 通過地点と休憩候補を把握して運行計画を立てたいから

## 5. 受け入れ条件

1. **Given** アプリ起動時 **When** 位置情報権限を許可 **Then** 現在地が地図上に表示される
2. **Given** 目的地をテキスト入力 **When** ルート検索完了 **Then** 一般道ルートと有料道路ルートが地図上に描画され、距離・所要時間が表示される
3. **Given** ナビ開始 **When** 運転開始 **Then** 430タイマーが自動的にカウントを開始する
4. **Given** 連続運転残り30分以下 **When** タイマー表示更新 **Then** パネルが赤色警告表示に切り替わる
5. **Given** ルート上にSA/PAがある **When** 休憩推薦を表示 **Then** 混雑度・空き台数・ETAが一覧で確認できる
6. **Given** ルート設定済み **When** ルートIC一覧を表示 **Then** IC名・距離が縦一列のタイムライン形式で確認できる
7. **Given** デモモード有効（standard） **When** アプリ操作 **Then** API接続なしで基本機能を体験できる

## 6. 非機能要件

| 種別 | 要件 | 備考 |
|------|------|------|
| 性能 | ルート検索は10秒以内に完了 | Google API タイムアウト10秒、OSRM フォールバック |
| 性能 | タイマー表示は5分単位で更新 | バッテリー消費抑制 |
| セキュリティ | API キーは .env で管理、バンドルに含めない | EXPO_PUBLIC_ プレフィックス使用 |
| 可用性 | Google API 障害時は OSRM / Nominatim にフォールバック | 2段構えの冗長化 |
| 可用性 | バックエンド未起動時はモックデータで動作 | 開発・デモ用 |
| 互換性 | iOS 15+ / Android 10+ | Expo SDK 54 対応範囲 |
| UX | DESIGN.md 準拠（JR東日本アプリ風、白背景 + #35A86E 緑） | DESIGN.md 参照 |

## 7. 依存・前提

- 外部サービス: Google Maps API（Directions / Places / Maps）、OSRM、Nominatim
- バックエンド: `EXPO_PUBLIC_API_BASE_URL` で指定するREST API（`/api/v1/navigation/recommend-rest`）
- ネイティブモジュール: `expo-location`、`react-native-maps`
- フラグ: `EXPO_PUBLIC_DEMO_MODE=true` でデモモード切替
- 注記: `@react-native-voice/voice`、`expo-speech` はMVPでは不使用（v0.2 で復帰予定）

## 8. 用語

| 用語 | 定義 |
|------|------|
| 430ルール | 連続運転4時間・休憩30分の法令基準（貨物自動車運送事業法） |
| デモモード | 外部API接続なしで固定シナリオを再生するモード |
| RouteTimeline | ルート上のIC・SA/PAを縦一列で表示するUIコンポーネント |
