## 1. 背景と目的

2024年4月に強化された「430ルール（連続運転4時間ごとに30分以上の休憩義務）」により、トラックドライバーは従来以上に厳密な時間管理と休憩場所確保を求められている。しかし現状では、ドライバーが適切なタイミングで適切な場所に休憩できるよう支援するツールが存在しない。

本機能は、連続運転時間のリアルタイム監視・QOL施設の動的検索・SA/PA混雑予測を組み合わせることで、法規制への受動的な対応を能動的な運行最適化に転換し、ドライバーの精神的・身体的負荷を軽減する。

## 2. スコープ

### In Scope

MVP優先度順：

1. **混雑回避・マッチング**（コア）— VICS情報および走行データに基づくSA/PA混雑予測と、空き施設へのドライバー分散誘導
2. **QOL施設検索** — 大型車駐車可能な施設（銭湯・24時間飲食店・コインランドリー・SA/PA）の地図検索・リコメンド
3. **430リアルタイム監視** — 連続運転時間のリアルタイム計測と残り走行可能時間の表示（5分単位更新）

共通基盤：

- オフライン環境（トンネル・山間部）での基本機能（時間計測・キャッシュ地図）の維持
- 1タップで操作可能なビッグボタンUI

### Out of Scope

- 運行管理会社向けの管理画面・ダッシュボード
- 荷物・配送ルートの管理機能
- 車両整備・点検記録
- 他ドライバーとのソーシャル機能
- 430ルール以外の労働規制（拘束時間・休息期間）の管理
- **音声操作**（走行中のハンズフリー入力・操作）— MVP対象外、将来フェーズで実装予定

## 3. ユーザーとゴール

| ユーザー | 達成したいこと |
|----------|----------------|
| 長距離トラックドライバー | 430ルールを違反せず、かつ運行効率を落とさずに業務を遂行したい |
| 長距離トラックドライバー | 快適に過ごせる休憩施設（風呂・食事・洗濯）をすぐに見つけたい |
| 長距離トラックドライバー | SA/PA満車による路肩停車リスクを避けたい |

## 4. ユースケース / ユーザーストーリー

1. **として** 長距離ドライバー **したい** 連続運転時間と残り走行可能時間を常に把握 **なぜなら** 430ルール違反による行政処分・事故リスクを避けるため
2. **として** 長距離ドライバー **したい** 現在地周辺の大型車対応QOL施設を即座に検索 **なぜなら** 知らない土地でも快適な休憩場所を確保するため
3. **として** 長距離ドライバー **したい** 目的地近くのSA/PA混雑状況を事前に把握 **なぜなら** 満車で駐車できず路肩停車を余儀なくされる状況を避けるため
4. **として** 長距離ドライバー **したい** 残り走行可能時間が少ない時点でアラートを受け取る **なぜなら** 適切な場所で余裕をもって休憩を開始するため
5. **として** 長距離ドライバー **したい** トンネルや山間部でもタイマーと地図が機能する **なぜなら** 電波が届かない区間でも時間管理を止めたくないため

## 5. 受け入れ条件

1. アプリ起動中、連続運転時間が5分ごとに更新され画面に表示される
2. 残り走行可能時間が30分・10分になった時点でそれぞれ通知が発火する
3. 現在地から半径指定（デフォルト20km）内の大型車対応施設が地図上に表示される
4. 施設フィルター（銭湯・飲食・コインランドリー・SA/PA）を1タップで切り替えられる
5. SA/PAの混雑レベル（空き・やや混雑・満車）が色分けアイコンで表示される
6. 機内モード相当のオフライン状態でも、タイマー計測とキャッシュ済み地図が動作する
7. 到着予想時刻の誤差が実測値に対して±5分以内である
8. 主要操作（タイマー開始/停止・施設検索）がホーム画面の1タップで完結する

## 6. 非機能要件

| 種別 | 要件 | 備考 |

| オフライン耐性 | 電波なし環境でタイマー計測・キャッシュ地図が動作すること | トンネル・山間部を想定 |
| 安全性 | 運転中の操作は最小限（通知はポップアップのみ、操作不要） | |
| 互換性 | iOS / Android 両対応 | |
| プライバシー | 位置情報は端末内処理を優先、外部送信する場合は明示同意を取得 | |

## 7. 依存・前提

- 外部サービス: VICS WIDE API（混雑情報取得）
- 外部サービス: 地図/施設検索 API（Google Maps Platform または国土地理院タイル + 独自DB）
- 前提: ユーザーが位置情報・通知の権限を許可していること
- 前提: 施設データ（大型車駐車可フラグ）の整備またはパートナーデータソースの確保

## 8. 用語

| 用語 | 定義 |
|------|------|
| 430ルール | 連続運転4時間ごとに30分以上の休憩を義務付ける道路交通法上の規制（2024年4月強化） |
| 連続運転時間 | 最後の休憩終了時点からの運転継続時間 |
| 残り走行可能時間 | 430ルール上、次の休憩義務発生までに走行できる残り時間 |
| QOLスポット | ドライバーの生活品質向上に資する施設（銭湯・24時間飲食店・コインランドリー等） |
| SA/PA | サービスエリア / パーキングエリア |
| VICS | 道路交通情報通信システム（Vehicle Information and Communication System） |

## 9. Open Questions

- 連続運転時間の計測基準：エンジンON/OFFか、速度センサーか、手動操作か
- 施設データソース：自社DB構築 vs. 既存サービス（トラックカーナビ等）との提携
- VICS API の利用コスト・レート制限の確認
- オフライン地図のキャッシュ戦略：事前ダウンロード範囲の定義
- 多重休憩（30分未満の複数休憩の合算）のカウント仕様
